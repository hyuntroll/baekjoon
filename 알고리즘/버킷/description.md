# Bucket (Binning) 기법
## 개요
Bucket (Binning) 기법 **데이터를 일정한 범위로 나누어 처리**하는 방식 입니다.

데이터를 **미리 분할하여 필요한 공간만 계산 또는 검색**하는 방법입니다.

### 주요 사용
- 범위 쿼리 문제
- 데이터 그룹화 문제

## 동작 원리
**Original: 1~1000개의 데이터**

| 1 | 2 | 3 | 4 | 5 | 6 | ... | 999 | 1000 |
|---|---|---|---|---|---|-----|-----|------|

**Bucket: 총 100개**

| 1~100 | 101~200 | 201~300 | ... | 901~1000 |
|-------|---------|---------|-----|----------|

### 1. 버킷 생성
데이터를 일정한 크기의 범위로 나눕니다.

> 예: 1000개의 데이터를 100개씩 나눕니다. 이때 범위가 **버킷**이 됩니다.

### 2. 데이터 분배

각 데이를 해당하는 버킷에 삽입합니다.

> 예: 453은 401~500에 해당하는 버킷에 삽입합니다.

### 3. 처리 수행

특정 범위의 데이터를 처리할 때, 해당 범위에 속한 버킷만 검사합니다.

> 예: 숫자 데이터를 정렬하는 대신 **버킷에 담고 필요한 범위만 탐색**하면 모든 데이터를 탐색하지 않아도 됩니다

## 시간 복잡도

- 버킷 생성 및 분배: $O(n)$
- 버킷 내 처리: 데이터 분포와 유형에 따라 다르지만 보통 $O(k+m)$
  - $k$: 탐색할 버킷 수
  - $m$: 버킷 내 데이터 수

**데이터가 고르게 분포**되어 있을 때 가장 효율적입니다.


## 장단점
### 장점
- 특정 범위의 데이터를 빠르게 검색 가능
- 모든 범위를 탐색하지 않음
- **큰 데이터 처리**에 효율적
- 다양한 문제에 적용 가능 (예: 정렬, 최솟값/최댓값 찾기, 검색)

### 단점
- **데이터가 불균등하게** 분포되면 비효율적
- 메모리 사용량 증가 가능

## 대표적인 코딩테스트 문제
- 범위 내 최대/최소값 찾기
- 주어진 범위의 데이터 합 계산
- 중복 데이터 탐색
- 값 정렬 또는 그룹화 문제

## 실제 사용 예시
- DB 인덱스: 범위 쿼리 최적화 (MySQL, PostgreSQL 파티셔닝)
- 해시 테이블: 해시 버킷이 정확히 이 구조
- 모니터링 툴: Prometheus의 histogram 메트릭이 버킷 단위로 데이터 집계
